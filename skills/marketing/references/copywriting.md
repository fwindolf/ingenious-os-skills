# Copywriting

Writing and improving marketing copy for ingeniOS pages.

## Core Principles

- Clarity > Cleverness
- Benefits > Features
- Specificity > Vagueness ("3 Minuten" not "schnell")
- Simple > Complex, Active > Passive, Confident > Qualified
- Customer language from `knowledge/product-marketing-context.md`

## Page Structure

Every page follows: Headline → Subheadline → CTA → Core Sections → Final CTA

### Headline Formulas
- "{Achieve outcome} ohne {pain point}" — "Rechnungen schreiben ohne Papierkram"
- "Nie wieder {unpleasant event}" — "Nie wieder vergessene Stunden"
- "{Key feature} für {audience}" — "Zeiterfassung für Handwerker"
- "Der/Die {category} der/die {differentiator}" — "Die Betriebssoftware die mitdenkt"

### Subheadline
Expand the headline — add specificity, mention the audience, or explain how.

### CTA Guidelines
- Action verb + what they get: "Kostenlos starten", "Jetzt testen", "Demo ansehen"
- Visible without scrolling
- One clear primary action per page
- Repeat at decision points (after social proof, after objection handling)

## Section Types

**Problem/Pain**: Describe their current frustration in their words. "Abends am Schreibtisch sitzen und Rechnungen schreiben — während die Familie wartet."

**Solution/How It Works**: 3-4 steps, simple. Show the transformation.

**Social Proof**: Testimonials (specific, attributed), logos, review scores. Place near CTAs.

**Objection Handling**: FAQ format. Address: price, complexity, data safety, switching effort.

**Feature → Benefit → Outcome**: Don't just list features. "ZUGFeRD-Rechnungen (feature) → automatisch rechtskonform (benefit) → kein Ärger mit dem Finanzamt (outcome)"

## Page-Specific

**Homepage**: Serve multiple personas (Meister, Bürokraft, Steuerberater). Start with broadest value prop, then segment.

**Landing Page**: Single message, single CTA. Match the traffic source exactly.

**Pricing Page**: Help choose, don't overwhelm. Highlight recommended tier. FAQ for objections.

**Feature Page**: Feature → benefit → outcome chain. Show the UI. Compare to status quo (Excel, paper).

## Writing Style for Handwerk

- Use: Betrieb, Meister, Baustelle, Stunden, Rechnung, Papierkram, Zettelwirtschaft, einfach
- Avoid: SaaS, Multi-Tenant, API, Onboarding, Workflow, Pipeline, Dashboard (use Übersicht)
- Short sentences. Concrete examples. German where possible.
